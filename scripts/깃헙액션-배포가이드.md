# GitHub Actions 자동 배포 가이드

GitHub에 코드를 푸시하면 자동으로 Docker 이미지를 빌드하고 Docker Hub에 배포하는 방법입니다.

## 🎯 장점

- ✅ 로컬 PC에 Docker 설치 불필요
- ✅ 코드 푸시만으로 자동 빌드/배포
- ✅ 버전 태그 자동 관리
- ✅ 빌드 로그 GitHub에서 확인 가능
- ✅ 무료 (GitHub Actions 무료 한도 내)

## 📋 초기 설정 (1회만)

### 1. GitHub Repository Secrets 설정

GitHub 저장소 → Settings → Secrets and variables → Actions → New repository secret

**추가할 Secrets:**

| Name | Value | 설명 |
|------|-------|------|
| `DOCKER_USERNAME` | `choho97` | Docker Hub 사용자명 |
| `DOCKER_PASSWORD` | `********` | Docker Hub 비밀번호 또는 Access Token |

**Docker Hub Access Token 생성 (권장):**
1. https://hub.docker.com/settings/security 접속
2. "New Access Token" 클릭
3. 토큰 이름: `github-actions`
4. 권한: `Read, Write, Delete`
5. 생성된 토큰을 `DOCKER_PASSWORD`로 사용

### 2. GitHub Workflow 파일 확인

파일이 자동 생성되었습니다:
```
.github/workflows/docker-build-push.yml
```

이 파일이 저장소에 있으면 자동으로 동작합니다.

## 🚀 사용 방법

### 방법 1: 코드 푸시로 자동 빌드 (latest)

```bash
git add .
git commit -m "Update backend code"
git push origin main
```

→ 자동으로 `choho97/aps-admin-backend:latest` 이미지 빌드 및 푸시

### 방법 2: 버전 태그로 릴리스 빌드

```bash
# 버전 태그 생성 및 푸시
git tag v1.0.0
git push origin v1.0.0
```

→ 자동으로 다음 이미지들이 빌드 및 푸시됨:
- `choho97/aps-admin-backend:1.0.0`
- `choho97/aps-admin-backend:latest`

### 방법 3: 수동 실행

GitHub 저장소 → Actions → "Docker Build and Push" → Run workflow

## 📊 빌드 상태 확인

### GitHub Actions 페이지
1. GitHub 저장소 → Actions 탭
2. 최근 워크플로우 실행 목록 확인
3. 실행 중인 작업 클릭 → 로그 확인

### 빌드 성공 확인
- ✅ 녹색 체크마크: 빌드 성공
- ❌ 빨간 X: 빌드 실패 (로그 확인)

### Docker Hub 확인
https://hub.docker.com/r/choho97/aps-admin-backend/tags

## 🔧 워크플로우 동작 방식

```yaml
트리거 조건:
1. main/master 브랜치에 push
2. v*.*.* 형식의 태그 push (예: v1.0.0)
3. 수동 실행 (workflow_dispatch)

빌드 단계:
1. 코드 체크아웃
2. Docker Buildx 설정
3. Docker Hub 로그인
4. 버전 추출 (태그 또는 latest)
5. 이미지 빌드 및 푸시
6. 빌드 캐시 저장 (다음 빌드 속도 향상)
```

## 🐛 문제 해결

### "Error: Invalid username/password"

**원인**: Docker Hub Secrets가 설정되지 않았거나 잘못됨

**해결**:
1. GitHub 저장소 → Settings → Secrets and variables → Actions
2. `DOCKER_USERNAME`과 `DOCKER_PASSWORD` 확인
3. Docker Hub 로그인 정보가 정확한지 확인

### "Error: denied: requested access to the resource is denied"

**원인**: Docker Hub 저장소 권한 없음

**해결**:
1. Docker Hub에서 `choho97/aps-admin-backend` 저장소 생성
2. Access Token 권한 확인 (Read, Write, Delete)

### 빌드가 시작되지 않음

**원인**: 워크플로우 파일 경로 또는 문법 오류

**확인**:
- 파일 경로: `.github/workflows/docker-build-push.yml`
- GitHub Actions 탭에서 에러 메시지 확인

### 빌드 시간이 너무 오래 걸림

**원인**: 첫 빌드는 캐시가 없어서 오래 걸림

**해결**:
- 두 번째 빌드부터는 캐시가 적용되어 빠름 (1-2분)
- 빌드 캐시: `choho97/aps-admin-backend:buildcache`

## 📂 로컬 빌드 vs GitHub Actions

| 항목 | 로컬 빌드 | GitHub Actions |
|------|----------|----------------|
| Docker 설치 | 필요 | 불필요 |
| 빌드 위치 | 로컬 PC | GitHub 서버 |
| 빌드 시간 | PC 성능에 따라 | 일정 (2-5분) |
| 로그 확인 | 터미널 | GitHub Actions |
| 자동화 | 수동 실행 | 푸시 시 자동 |
| 비용 | 무료 | 무료 (한도 내) |

## 🔐 보안

### Secrets 관리
- ✅ GitHub Secrets는 암호화되어 저장
- ✅ 로그에 비밀번호가 표시되지 않음
- ✅ Access Token 사용 권장 (비밀번호보다 안전)

### 이미지 내용
- ✅ `.dockerignore`로 민감 정보 제외
- ✅ `service-account.json` 제외됨
- ✅ `.env` 파일 제외됨
- ✅ 소스코드만 포함

## 📊 GitHub Actions 무료 한도

**Public Repository:**
- 무제한 빌드 시간

**Private Repository:**
- 월 2,000분 무료
- 1회 빌드 약 2-5분 소요
- 월 약 400-1000회 빌드 가능

## 🛠️ 고급 설정

### 특정 브랜치만 빌드

`.github/workflows/docker-build-push.yml` 수정:
```yaml
on:
  push:
    branches:
      - main
      - develop  # develop 브랜치 추가
```

### 빌드 조건 추가

특정 파일 변경 시만 빌드:
```yaml
on:
  push:
    branches:
      - main
    paths:
      - 'backend/**'  # backend 폴더 변경 시만
```
