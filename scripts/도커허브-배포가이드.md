# Docker Hub 백엔드 이미지 배포 가이드

APS Admin 백엔드는 GitHub Actions가 아니라 로컬/서브컴에서 Docker 이미지를 직접 빌드해서 Docker Hub에 올리고, NAS가 해당 태그를 pull해서 실행합니다.

## 원칙

- 운영 배포에는 `latest`를 쓰지 않습니다.
- 백엔드 이미지는 반드시 `1.3.2` 같은 구체적인 태그로 빌드합니다.
- NAS의 `nas-deploy/.env`에는 현재 운영 태그를 `BACKEND_IMAGE_TAG=1.3.2`처럼 기록합니다.
- `aligo-proxy`는 SMS처럼 고정 IP가 필요한 기능만 담당합니다. 일반 앱 API/WebSocket 트래픽은 Cloudflare Tunnel을 통해 NAS 백엔드로 직결합니다.

## 백엔드 이미지 빌드/푸시

### Windows PowerShell

```powershell
# 예: choho97/aps-admin-backend:1.3.2 빌드/푸시
.\scripts\docker-build-push.ps1 -Version 1.3.2
```

`latest` 태그도 같이 갱신해야 할 때만 명시합니다. 운영 배포 기준은 여전히 구체 태그입니다.

```powershell
.\scripts\docker-build-push.ps1 -Version 1.3.2 -PushLatest
```

### Linux/Mac

```bash
chmod +x scripts/docker-build-push.sh

# 예: choho97/aps-admin-backend:1.3.2 빌드/푸시
./scripts/docker-build-push.sh 1.3.2
```

`latest` 태그도 같이 갱신해야 할 때:

```bash
./scripts/docker-build-push.sh 1.3.2 --push-latest
```

## NAS에 배포

권장 방식은 로컬에서 NAS 배포 스크립트를 실행하는 것입니다.

```powershell
# 먼저 실제 변경 없이 확인
.\scripts\deploy-nas-backend.ps1 -BackendImageTag 1.3.2 -SyncCompose -WhatIf

# 확인 후 배포
.\scripts\deploy-nas-backend.ps1 -BackendImageTag 1.3.2 -SyncCompose
```

스크립트가 하는 일:

1. NAS의 `nas-deploy/.env`를 백업합니다.
2. `BACKEND_IMAGE_TAG=1.3.2`를 설정합니다.
3. `WS_RELAY_ENABLED=false`, `BACKEND_ENVIRONMENT=production`을 보장합니다.
4. 필요 시 `nas-deploy/docker-compose.yml`을 NAS로 동기화합니다.
5. `docker compose pull aps-backend` 후 `docker compose up -d`를 실행합니다.
6. `http://localhost:3001/` health check를 확인합니다.

## 수동 배포가 필요할 때

NAS에서 직접 실행:

```bash
cd /srv/dev-disk-by-uuid-e7c1c262-c767-41e3-aeab-ab5a786ac5a1/docker-nas

cp .env ".env.backup.$(date +%Y%m%d-%H%M%S)"
grep -q '^BACKEND_IMAGE_TAG=' .env \
  && sed -i 's/^BACKEND_IMAGE_TAG=.*/BACKEND_IMAGE_TAG=1.3.2/' .env \
  || echo 'BACKEND_IMAGE_TAG=1.3.2' >> .env

grep -q '^WS_RELAY_ENABLED=' .env \
  && sed -i 's/^WS_RELAY_ENABLED=.*/WS_RELAY_ENABLED=false/' .env \
  || echo 'WS_RELAY_ENABLED=false' >> .env

docker compose pull aps-backend
docker compose up -d
curl http://localhost:3001/
```

## 보안 주의

- `.env`와 `service-account.json`은 이미지에 포함하지 않습니다.
- NAS에는 `service-account.json`을 로컬 파일로 마운트합니다.
- SMS/Aligo 관련 키는 `.env`에만 둡니다.
- `latest`는 캐시와 롤백 추적이 애매해지므로 운영 배포 기준으로 쓰지 않습니다.
