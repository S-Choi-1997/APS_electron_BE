## 📦 다른 서버에서 사용하기

### 1. 이미지 다운로드

```bash
# Docker Hub에서 이미지 다운로드
docker pull choho97/aps-admin-backend:latest

# 특정 버전 다운로드
docker pull choho97/aps-admin-backend:1.0.0
```

### 2. 필수 파일 준비

다운로드받은 서버에 다음 파일들이 필요합니다:

```
서버-디렉토리/
├── .env                    # 환경 변수 (백엔드 설정)
├── service-account.json    # GCP 서비스 계정 키
└── docker-compose.yml      # (선택) Docker Compose 설정
```

**`.env` 파일** ([.env.example](../backend/.env.example) 참고):
```env
PORT=3001
ALLOWED_ORIGINS=http://localhost:5173,app://aps-admin
ALLOWED_EMAILS=admin@example.com
STORAGE_BUCKET=aps-list.appspot.com
GOOGLE_APPLICATION_CREDENTIALS=/app/service-account.json
NAVER_CLIENT_ID=...
NAVER_CLIENT_SECRET=...
ALIGO_API_KEY=...
ALIGO_USER_ID=...
ALIGO_SENDER_PHONE=...
RELAY_URL=http://136.113.67.193:3000
```

**`service-account.json`**: GCP 서비스 계정 JSON 파일

### 3. 실행 방법

#### 방법 A: docker run (간단)

```bash
docker run -d \
  --name aps-admin-backend \
  --restart unless-stopped \
  -p 3001:3001 \
  --env-file .env \
  -v $(pwd)/service-account.json:/app/service-account.json:ro \
  choho97/aps-admin-backend:latest
```

**Windows (PowerShell)**:
```powershell
docker run -d `
  --name aps-admin-backend `
  --restart unless-stopped `
  -p 3001:3001 `
  --env-file .env `
  -v ${PWD}/service-account.json:/app/service-account.json:ro `
  choho97/aps-admin-backend:latest
```

#### 방법 B: docker-compose (권장)

**docker-compose.yml** 생성:

```yaml
version: '3.8'

services:
  aps-backend:
    image: choho97/aps-admin-backend:latest
    container_name: aps-admin-backend
    restart: unless-stopped
    ports:
      - "3001:3001"
    env_file:
      - .env
    volumes:
      - ./service-account.json:/app/service-account.json:ro
    environment:
      GOOGLE_APPLICATION_CREDENTIALS: /app/service-account.json
    networks:
      - aps-network

networks:
  aps-network:
    driver: bridge
```

실행:
```bash
docker-compose up -d
```

## 🔄 업데이트 방법

새 버전을 배포한 후, 다른 서버에서 업데이트:

```bash
# 1. 새 이미지 다운로드
docker pull choho97/aps-admin-backend:latest

# 2. 기존 컨테이너 중지 및 삭제
docker stop aps-admin-backend
docker rm aps-admin-backend

# 3. 새 컨테이너 시작
docker-compose up -d
# 또는
docker run -d --name aps-admin-backend ... (위의 명령어 사용)
```

**docker-compose 사용 시 더 간단**:
```bash
docker-compose pull   # 새 이미지 다운로드
docker-compose up -d  # 자동으로 재시작
```

## 🔐 보안 주의사항

### Docker Hub 저장소 설정

1. **Public vs Private**:
   - Public: 무료, 누구나 다운로드 가능 (소스코드만 공개, 민감정보 없음)
   - Private: 유료 (무료 플랜: Private 저장소 1개), 인증 필요

2. **민감 정보**:
   - ❌ `.env` 파일은 이미지에 포함되지 않음
   - ❌ `service-account.json`은 이미지에 포함되지 않음
   - ✅ 소스코드(`server.js`)만 포함됨

3. **이미지에 포함되는 것**:
   ```
   ✅ server.js (소스코드)
   ✅ package.json, package-lock.json
   ✅ node_modules (빌드 시 설치)
   ❌ .env (제외됨)
   ❌ service-account.json (제외됨)
   ❌ README.md, setup scripts (제외됨)
   ```

4. **`.dockerignore` 확인**:
   ```
   .env
   .env.*
   service-account.json
   service-account.json.backup.*
   node_modules
   npm-debug.log
   .git
   .gitignore
   README.md
   docker-compose.yml
   ```

## 🌐 네트워크 설정

### OMV NAS / 로컬 서버에서 실행 시

1. **방화벽 포트 개방**:
   ```bash
   sudo ufw allow 3001/tcp
   ```

2. **프론트엔드 `.env.production` 설정**:
   ```env
   VITE_API_URL=http://192.168.0.100:3001  # NAS 서버 IP
   ```

3. **CORS 설정 확인** (백엔드 `.env`):
   ```env
   ALLOWED_ORIGINS=http://localhost:5173,app://aps-admin
   ```

## 📝 트러블슈팅

### "docker: permission denied"

Linux/Mac에서 sudo 없이 docker 사용:
```bash
sudo usermod -aG docker $USER
# 로그아웃 후 다시 로그인
```

### "unauthorized: authentication required"

Docker Hub 로그인 필요:
```bash
docker logout
docker login
```

### "repository does not exist or may require 'docker login'"

- Docker Hub에 저장소가 존재하는지 확인
- `DOCKER_USERNAME`이 올바른지 확인
- Private 저장소인 경우 로그인 확인
