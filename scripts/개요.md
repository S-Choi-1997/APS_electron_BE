# 개발 스크립트 및 문서

이 폴더에는 APS Admin 프로젝트의 개발 환경 관리를 위한 보조 파일들이 포함되어 있습니다.

## 📁 파일 목록

### 개발 환경 스크립트

- **start-dev.ps1** - 개발 환경 전체 실행 (백엔드 + Vite + Electron)
- **stop-dev.ps1** - 실행 중인 모든 개발 서비스 중지
- **check-infra.ps1** - NAS/aligo-proxy 상태 읽기 전용 확인
- **build-app-release.ps1** - Cloudflare 백엔드 URL로 Electron 설치 파일 빌드
- **deploy-nas-backend.ps1** - NAS에서 지정한 백엔드 이미지 태그 pull/up

### Docker Hub 배포 스크립트

- **docker-build-push.ps1** - 백엔드 Docker 이미지 빌드 및 Docker Hub 푸시 (Windows, concrete tag 필수)
- **docker-build-push.sh** - 백엔드 Docker 이미지 빌드 및 Docker Hub 푸시 (Linux/Mac, concrete tag 필수)

### 문서

- **개발환경-실행가이드.md** - 개발 환경 설정 및 실행 가이드
- **도커허브-배포가이드.md** - Docker Hub 빌드/배포 가이드 (로컬)

## 🚀 사용법

### 개발 환경 시작

```powershell
.\scripts\start-dev.ps1
```

또는 프로젝트 루트에서:

```powershell
cd E:\Projects\APS\APS_APP
.\scripts\start-dev.ps1
```

### 개발 환경 중지

```powershell
.\scripts\stop-dev.ps1
```

### 운영 상태 확인

```powershell
.\scripts\check-infra.ps1 -BackendUrl https://your-cloudflare-backend-domain
```

### 앱 빌드

```powershell
.\scripts\build-app-release.ps1 -BackendUrl https://your-cloudflare-backend-domain
```

### 백엔드 이미지 빌드/푸시

```powershell
# Windows
.\scripts\docker-build-push.ps1 -Version 1.3.2

# Linux/Mac
./scripts/docker-build-push.sh 1.3.2
```

자세한 내용은 [도커허브-배포가이드.md](도커허브-배포가이드.md)를 참고하세요.

### NAS 백엔드 배포

```powershell
.\scripts\deploy-nas-backend.ps1 -BackendImageTag 1.3.2 -SyncCompose
```

## 📌 참고사항

- PowerShell 실행 정책 오류 발생 시:
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```

- 스크립트는 프로젝트 루트 디렉토리에서 실행되도록 설계되었습니다.
- 3개의 별도 PowerShell 창이 열립니다 (백엔드, Vite, Electron)
