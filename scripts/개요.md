# 개발 스크립트 및 문서

이 폴더에는 APS Admin 프로젝트의 개발 환경 관리를 위한 보조 파일들이 포함되어 있습니다.

## 📁 파일 목록

### 개발 환경 스크립트

- **start-dev.ps1** - 개발 환경 전체 실행 (백엔드 + Vite + Electron)
- **stop-dev.ps1** - 실행 중인 모든 개발 서비스 중지

### Docker Hub 배포 스크립트

- **docker-build-push.ps1** - Docker 이미지 빌드 및 Docker Hub 푸시 (Windows)
- **docker-build-push.sh** - Docker 이미지 빌드 및 Docker Hub 푸시 (Linux/Mac)

### 문서

- **개발환경-실행가이드.md** - 개발 환경 설정 및 실행 가이드
- **도커허브-배포가이드.md** - Docker Hub 빌드/배포 가이드 (로컬)
- **깃헙액션-배포가이드.md** - GitHub Actions 자동 빌드/배포 가이드 (권장)

## 🚀 사용법

### 개발 환경 시작

```powershell
.\scripts\start-dev.ps1
```

또는 프로젝트 루트에서:

```powershell
cd E:\Projects\APS\APS_APP
.\scripts\start-dev.ps1
```

### 개발 환경 중지

```powershell
.\scripts\stop-dev.ps1
```

### Docker Hub에 배포

#### GitHub Actions (권장)
```bash
# 코드 푸시만으로 자동 빌드
git push origin main

# 버전 릴리스
git tag v1.0.0
git push origin v1.0.0
```

자세한 내용은 [깃헙액션-배포가이드.md](깃헙액션-배포가이드.md)를 참고하세요.

#### 로컬 빌드 (Docker 설치 필요)
```powershell
# Windows
.\scripts\docker-build-push.ps1 1.0.0

# Linux/Mac
./scripts/docker-build-push.sh 1.0.0
```

자세한 내용은 [도커허브-배포가이드.md](도커허브-배포가이드.md)를 참고하세요.

## 📌 참고사항

- PowerShell 실행 정책 오류 발생 시:
  ```powershell
  Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
  ```

- 스크립트는 프로젝트 루트 디렉토리에서 실행되도록 설계되었습니다.
- 3개의 별도 PowerShell 창이 열립니다 (백엔드, Vite, Electron)
