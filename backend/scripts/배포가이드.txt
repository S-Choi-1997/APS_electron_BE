# APS backend deployment guide

Current backend deployment uses a Docker Hub image on the NAS/backend server.
The deployment server does not build backend source code.

## 1. Build and push a backend image when backend code changes

Build from the project root:

```bash
# Windows
.\scripts\docker-build-push.ps1 1.3.1

# Linux/Mac
./scripts/docker-build-push.sh 1.3.1
```

This publishes:

```text
choho97/aps-admin-backend:1.3.1
```

Use concrete version tags for rollback control. Avoid `latest` for normal production updates.

## 2. Backend server files

The NAS/backend server uses `nas-deploy/`:

```text
nas-deploy/
├── docker-compose.yml
├── init-db.sql
├── .env                 # create manually, do not commit
└── service-account.json # copy manually, do not commit
```

`nas-deploy/docker-compose.yml` runs:

```text
choho97/aps-admin-backend:${BACKEND_IMAGE_TAG}
```

## 3. Set `nas-deploy/.env`

Important values:

```env
BACKEND_IMAGE_TAG=1.3.1
WS_RELAY_ENABLED=false
BACKEND_ENVIRONMENT=production
JWT_SECRET=<random 64 hex characters>
JWT_REFRESH_SECRET=<different random 64 hex characters>
SMS_RELAY_URL=http://136.113.67.193:3000
```

SMS still uses `SMS_RELAY_URL`. Normal Electron app traffic does not use the old app relay.

## 4. Deploy on the backend server

```bash
cd nas-deploy
docker-compose pull aps-backend
docker-compose up -d
docker-compose logs -f aps-backend
```

Health check:

```bash
curl http://localhost:3001/
```

Expected fields:

```json
{
  "wsRelayEnabled": false,
  "directWebSocket": {
    "enabled": true
  }
}
```

## 5. Cloudflare check

Cloudflare Tunnel should forward the public backend hostname to `http://localhost:3001` on the backend server.

```bash
curl https://your-cloudflare-backend-domain/
```

The Electron app release should use:

```env
VITE_API_URL=https://your-cloudflare-backend-domain
VITE_BACKEND_ENVIRONMENT=production
```
