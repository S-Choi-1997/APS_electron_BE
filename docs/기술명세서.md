## 4. 기능 목록

### 4.1 인증/계정

| 기능 | 구현 방식 |
|------|-----------|
| 이메일/비밀번호 로그인 | JWT (Access 1h + Refresh 30d) |
| 자동 로그인 | Refresh Token을 localStorage에 저장 후 앱 시작 시 자동 복원 |
| 이메일 저장 | localStorage에 마지막 로그인 이메일 저장 |
| 로그아웃 | Refresh Token 서버 폐기 + localStorage 클리어 |
| 토큰 자동 갱신 | API 호출 시 401 응답 시 Refresh Token으로 자동 갱신 후 재요청 |
| 계정 생성 | CLI 스크립트 (create-admin.js) |
| 역할 관리 | admin / user (Firestore admins 컬렉션) |
| 속도 제한 | 로그인 5회/15분 (IP 기반) |

### 4.2 홈페이지 상담문의 관리

| 기능 | 구현 방식 |
|------|-----------|
| 문의 목록 조회 | Firestore 기반, 필터링 (상태/카테고리/체크 여부) |
| 문의 상세 조회 | 모달 팝업 |
| 상태 변경 | unread → read → responded |
| 체크 표시 | 개별 체크 + 전체 선택 |
| 첨부파일 다운로드 | GCS Signed URL 생성 → Electron 파일 저장 대화상자 |
| SMS 답변 | Aligo API 경유 문자 발송 |
| 카테고리 분류 | visa, nonprofit, corporate, civil, etc |
| 페이지네이션 | limit/offset 기반 |
| 검색 | 디바운스 검색 (300ms) |
| 실시간 수신 알림 | Firestore onSnapshot → WebSocket → 토스트 알림 |

### 4.3 이메일 문의 관리

| 기능 | 구현 방식 |
|------|-----------|
| 이메일 목록 조회 | PostgreSQL, 필터링 (source/status/check) |
| ZOHO Mail 동기화 | OAuth 2.0 인증 → API 전체/증분 동기화 |
| ZOHO Webhook 수신 | HMAC SHA256 서명 검증 → 실시간 수신 |
| 이메일 답장 | ZOHO Mail API로 회신 발송 |
| 이메일 스레드 | inReplyTo 기반 대화 스레드 추적 |
| 첨부파일 관리 | ZOHO API로 첨부파일 목록 조회/다운로드 |
| 소스별 분류 | gmail / zoho 구분 |
| 통계 조회 | total, unread, read, responded, gmail, zoho 카운트 |
| 자동 동기화 | 15분 주기 서버 크론 |

### 4.4 메모 관리

| 기능 | 구현 방식 |
|------|-----------|
| 메모 CRUD | PostgreSQL 기반 |

```sql
created_at      TIMESTAMP DEFAULT NOW()
```

```sql
id              SERIAL PRIMARY KEY
title           VARCHAR(255) NOT NULL
content         TEXT
important       BOOLEAN DEFAULT FALSE
author          VARCHAR(255) NOT NULL    -- 작성자 이메일
author_name     VARCHAR(255)
expire_date     TIMESTAMP
created_at      TIMESTAMP DEFAULT NOW()
updated_at      TIMESTAMP DEFAULT NOW()

```

```sql
id              SERIAL PRIMARY KEY
title           VARCHAR(255) NOT NULL

start_date      DATE NOT NULL
end_date        DATE
type            VARCHAR(50)              -- 'company' | 'personal'
author          VARCHAR(255) NOT NULL
author_name     VARCHAR(255)
created_at      TIMESTAMP DEFAULT NOW()
updated_at      TIMESTAMP DEFAULT NOW()
deleted_at      TIMESTAMP                -- 소프트 삭제
```

#### email_inquiries
```sql
id              SERIAL PRIMARY KEY
from_address    VARCHAR(255)
subject         TEXT
message_id      VARCHAR(255) UNIQUE      -- 중복 방지
body            TEXT
status          VARCHAR(50) DEFAULT 'unread'  -- unread | read | responded
check_status    BOOLEAN DEFAULT FALSE
is_outgoing     BOOLEAN DEFAULT FALSE
source          VARCHAR(50)              -- 'gmail' | 'zoho'
in_reply_to     VARCHAR(255)             -- 스레드 추적
```

```sql
id              SERIAL PRIMARY KEY
```

## 9. 보안 구현 사항

| 항목 | 구현 방식 |
|------|-----------|
| 비밀번호 해싱 | bcrypt 12 라운드 |
| JWT Access Token | 1시간 만료, HS256 서명 |
| JWT Refresh Token | 30일 만료, 롤링 갱신, DB 저장 |
| CORS | Origin 화이트리스트 (환경변수) |
| 속도 제한 | express-rate-limit (로그인 5회/15분) |
| XSS 방지 | DOMPurify로 HTML 새니타이징 |
| SQL Injection 방지 | pg 파라미터화 쿼리 ($1, $2) |
| 웹훅 검증 | HMAC SHA256 서명 + 상수시간 비교 |
| Context Isolation | Electron contextIsolation: true |
| Node Integration 비활성화 | nodeIntegration: false |
| IPC 브릿지 | contextBridge로 안전한 API 노출 |

## 12. 데이터 흐름도

### 12.1 인증 흐름
```
[사용자] → 이메일/비밀번호 입력
  → [LoginPage.jsx] → localAuth.signInWithLocal()
    → POST /auth/login (rate limited)
      → [auth.js] Firestore admins 조회 → bcrypt 비교
        → JWT 생성 (access + refresh)
        → PostgreSQL refresh_tokens 저장
      ← {user, accessToken, refreshToken}
    → localStorage 저장 ('aps-local-auth-user')
    → authManager 상태 업데이트
    → WebSocket 연결 시작 (handshake)
  ← 대시보드 표시
```

### 12.2 자동 로그인 흐름
```
[앱 시작]
  → authManager.restoreSession()
    → localStorage 'aps-auto-login' 확인
    → localStorage 'aps-local-auth-user' 에서 refreshToken 추출
      → POST /auth/refresh
        → JWT 검증 → 새 토큰 쌍 발급
        → 이전 refreshToken DB 삭제 → 새 토큰 저장
      ← 새 {accessToken, refreshToken}
    → localStorage 갱신
    → 인증 완료
```

### 12.3 실시간 동기화 흐름
```
[외부 이벤트] (웹폼 제출 / ZOHO 웹훅 / API 호출)
  → [Backend server.js]
    → 데이터 처리 (Firestore/PostgreSQL)
    → broadcastEvent(eventType, data)
      → [WebSocket Relay (GCP VM:8080)]
        → 전체 연결 클라이언트에 브로드캐스트
          → [Electron Main Process]
            → broadcastToAllWindows(eventName, eventData)
              → [Renderer: useWebSocketSync 훅]
                → React Query 캐시 무효화/업데이트
                  → UI 자동 리렌더링
```

### 12.4 API 요청 흐름 (프로덕션)
```
[Renderer]
  → apiRequest() (src/config/api.js)
    → Authorization: Bearer {accessToken} 헤더 첨부
    → X-Relay-Environment: production 헤더 첨부
      → HTTP → [WebSocket Relay:8080/proxy]
        → http:request 이벤트 → [Backend]
          → authenticateJWT 미들웨어
            → JWT 검증 → req.user 설정
          → 라우트 핸들러 실행
          → 응답 생성
        ← http:response 이벤트 → [Relay]
      ← HTTP 응답 → [Renderer]
    → 401 시: localAuth.restoreSession() → 재요청
```
